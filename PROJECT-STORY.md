## 1. 项目起源

**一句话**: 做一个能识别初学者为什么错、并一步步引导他们真正理解 Python 的 AI 导师。

不是做一个"Python 版 ChatGPT"，而是做一个真正的 **AI Tutor**：

- ❌ 不会直接给答案 → ✅ 用分层提示引导学生思考
- ❌ 只说"代码错了" → ✅ 诊断背后的认知误区
- ❌ 千人一面 → ✅ 四个入口匹配不同基础

技术选型：**FastAPI**（后端）+ **Next.js 16**（前端）+ **DeepSeek**（AI）+ **SQLite**（数据库）。

## 2. 骨架搭建（Phase 1-6）

### 2.1 后端：FastAPI + SQLAlchemy + JWT

**做了什么**：

1. 用 FastAPI 搭建 Web 服务，Uvicorn 作为服务器
2. SQLAlchemy 2.0 异步操作数据库，开发用 SQLite，生产可切 PostgreSQL
3. bcrypt 加密密码，JWT 做登录令牌
4. structlog 做结构化日志（开发时彩色输出，生产时 JSON）

**为什么这样选**：

FastAPI 是 Python 最快的 Web 框架之一，支持异步。SQLAlchemy 2.0 的异步模式能同时处理多个请求而不互相等待。JWT 令牌让前端记住登录状态，不用每次都输密码。

### 2.2 前端：Next.js + Tailwind CSS

**做了什么**：

1. Next.js 16 App Router + React 19
2. Tailwind CSS 4 做暗色主题
3. Zustand 管理登录状态、聊天状态、练习状态
4. Monaco Editor 作为代码编辑器

### 2.3 AI 对话

**怎么工作**：

```
用户输入消息 → FastAPI 接收
  → 获取历史对话（最近 20 条）
  → RAG 检索相关知识点
  → 拼成 Prompt 发给 DeepSeek
  → 解析 AI 回复 → 返回给前端
```

**关键代码**：`chat_service.py` 的 `send_message()` 函数，7 个步骤完成从接收消息到返回 AI 回复。

### 2.4 代码实验室（Code Lab）

用户写 Python 代码 → Docker 沙箱安全执行 → 返回 stdout/stderr。如果代码有问题，AI 自动分析错误原因。

### 2.5 练习中心（ACM 模式）

- AI 生成练习题
- 用户提交代码 → 自动运行测试用例
- 判题结果：Accepted / Wrong Answer
- 提示系统：点击"提示"按钮逐步获取帮助

## 3. RAG 语义检索：从崩溃到重生（Phase 7）

### 问题：聊天卡死 485 秒

**现象**：用户发一条消息"1+1等于几"，等了 8 分钟才收到回复。

**根因**：RAG 系统使用了一个叫 `sentence-transformers` 的本地 AI 模型来做语义理解。这个模型 100MB，需要从 HuggingFace 网站下载。但在中国，HuggingFace 被墙了。Python 代码是同步的——下载时整个程序卡住不动，一直等到超时（485 秒）。

**教训**：在异步 Python 中，任何涉及网络下载的操作都必须放到后台线程。

### 解决方案：DashScope API

**为什么选它**：项目已经在用阿里云百炼的 DashScope 做图片识别，同一个平台有文本向量 API（`text-embedding-v3`）。

**怎么工作**：把文字发给 DashScope API → 返回 1024 个数字（向量）→ 存入 ChromaDB 数据库 → 用户提问时搜索最相似的片段。

**效果**：
- 响应时间：485 秒 → 3-4 秒
- 向量维度：384 → 1024（更精确）
- 维护：零（不需要下载模型）

**新发现的问题**：DashScope 每批最多 10 条文本，不是文档说的 25 条。此外还有个隐蔽 Bug——`db.execute()` 返回的结果被第一个循环用完后，第二个循环拿到的就是空列表。

## 4. PyTutor 2.0：从 Chatbot 到 Tutor（Phase 8）

这是项目最大的升级。之前 PyTutor 只是一个"能用 AI 回答 Python 问题"的网站。2.0 的目标是让它成为一个"能识别学生为什么错，并一步步引导"的 AI 导师。

### 4.1 误区诊断系统（Misconception Diagnosis）

**什么是误区**：初学者常犯的概念错误。比如：
- M1：把 `=`（赋值）当成 `==`（比较）用
- M3：以为 `list.append()` 会返回新列表

**怎么诊断**：双重引擎
1. **规则匹配**（快速）：用正则表达式扫描代码，匹配已知错误模式
2. **LLM 辅助**（准确）：规则不确定时，让 AI 帮忙判断

### 4.2 教学策略系统（Pedagogy Engine）

**核心原则**：不要直接给答案。

```
诊断出误区 → 判断是第几次出现 → 选择策略
  第 1 次 → 渐进提示（Level 1-2，只给概念提示）
  第 3+ 次 → 概念解释（停下来讲清楚这个知识点）
  已修正 → 总结反思
```

**5 级提示**：
- Level 1："你对代码的运行过程可能有误解，先观察一下输出"
- Level 5：给出完整参考答案（只在学生多次尝试失败后）

### 4.3 AI 回复质量自检

AI 生成回复后，系统自己检查一遍：有没有过早给答案？是否符合当前提示等级？不合格就重新生成。

## 5. 新手教程系统（Phase 9-10）

### 5.1 四入口设计

根据基础水平分四个路径：
- **A. 零基础** → 12 课完整教程（从认识编辑器到综合项目）
- **B. 学过一点** → 6 题诊断 → 按结果推荐补漏或练习
- **C. 会基础** → 直接进练习中心
- **D. 自由提问** → AI 对话

### 5.2 LessonPlayer 组件

通用课程播放器，课程内容和数据逻辑分离：
- 左侧：AI Tutor 分步引导
- 右侧：代码编辑器 + 运行 + 输出
- 底部：进度条 + 下一课按钮

添加新课只需在 `tutorial-data.ts` 里加数据，不用改组件代码。

### 5.3 教师仪表盘

展示全平台的教学数据：
- 误区频次排行
- 薄弱知识点 TOP 10
- 提示依赖度分布
- 学生列表 + 最近动态

### 5.4 误区专项练习

16 道靶向练习题（M1-M8 每种误区 2 道），含"错误修复"和"输出预测"两种类型。

## 6. 部署之旅：让全世界能访问

这是项目中最曲折的部分。目标是让朋友通过浏览器 URL 就能使用 PyTutor，不花钱。

### 6.1 方案选择

| 组件 | 平台 | 费用 |
|------|------|------|
| 后端 FastAPI | Render | 免费 750h/月 |
| 前端 Next.js | Vercel | 免费 |

### 6.2 后端部署：Render（踩坑 10+ 次）

**踩坑 1**：Render 不读 `render.yaml`
- 手动设 Root Directory = `backend`
- 手动设 Start Command = `uvicorn app.main:app`

**踩坑 2**：`requirements.txt` 版本问题
- 本地 `litellm==1.88.1` 在 Render 的 PyPI 镜像上不存在
- 解决：去掉所有版本号，让 pip 自动匹配

**踩坑 3**：重包安装失败导致后续包丢失
- `chromadb` 和 `sentence-transformers` 安装失败后，pip 停止安装后续包
- `litellm` 排在后面，没装上 → 语法错误又让部署卡死
- 解决：删服务重建，精简依赖，添加 openai SDK 备用通道

**踩坑 4**：部署队列卡死无法取消
- Render 免费版没 Cancel 按钮
- 唯一的出路：删除整个服务，重建

**踩坑 5**：语法错误导致启动崩溃
- `llm_service.py` 有两个连续的 `except` 块
- 解决：本地 `py_compile` 验证语法后再推送

### 6.3 前端部署：Vercel（踩坑 8+ 次）

**踩坑 1**：Root Directory 没设 `frontend`
- Vercel 在项目根目录找 `package.json`，找不到报错
- 解决：Import 项目时设置 Root Directory = `frontend`

**踩坑 2**：Git 自动部署权限问题
- Git 作者邮箱 `dev@pytutor.local` 与 Vercel 账号不匹配
- 解决：改 Git 配置为 GitHub 邮箱

**踩坑 3**：CLI 部署反复失败
- `vercel.json` 被自动生成，导致项目名冲突
- 解决：删除 Vercel 项目，全新 Import

**踩坑 4**：SSR 时 `localStorage` 不存在
- Next.js 在服务器端渲染时，`localStorage` 不可用
- 解决：添加 `typeof window === "undefined"` 守卫

**踩坑 5**：页面不停闪烁
- 3 个原因叠加：
  1. `api.ts` 401 时强制 `window.location.href` 跳转
  2. `loadUser` 失败时清零 `isAuthenticated`
  3. OnboardingWrapper 等待后端 API 响应时阻塞 UI
- 解决：本地优先判断（token + localStorage），删除所有强制跳转

### 6.4 最终成果

| 层 | 地址 |
|------|------|
| 前端 | `https://pytutor-ai-delta.vercel.app` |
| 后端 | `https://pytutor-ai-6tq0.onrender.com` |

### 部署问题（#20, #27, #30-#35）

| Bug | 教训 |
|-----|------|
| #20 AI 卡死 485s | HuggingFace 在国内不可达 |
| #27 管理员 SQLite 直插无法登录 | 永远通过 API 创建账号 |
| #30 页面不停闪烁 | 不要把后端依赖写进 UI 渲染路径 |
| #31 退出后弹窗残留 | useEffect 依赖要完整 |
| #32 SSR localStorage 崩溃 | 服务端渲染时检查 `typeof window` |
| #33-#35 Vercel/Render 部署连环失败 | 云部署的依赖、权限、语法每一个环节都要预检 |

### 逻辑问题（#21-#24, #26, #28-#29）

| Bug | 教训 |
|-----|------|
| #21 向量索引写入 0 条 | Python 迭代器只能消费一次 |
| #22 生成题目就显示对勾 | 完成状态必须用唯一 ID，不能用分类标签 |
| #23 练习完成数翻倍 | 计数器只能有一个权威来源 |
| #24 做对了没打勾 | `already_passed` 应按题目 ID 去重 |
| #26 新用户看到旧进度 | localStorage key 必须绑定用户 ID |
| #28-#29 教师看到学生弹窗 | 角色检查要等 user 对象加载完 |
